# AI Work Split

## Principle

AI should operate the system, not define the truth of the product.

That distinction matters.

If AI invents user truth or business logic, the product becomes noisy and untrustworthy.

If AI is used to transform, enrich, score, and explain structured data, it becomes useful.

## What AI Should Do

### 1. Data Enrichment

AI can:

- normalize titles
- infer tags from natural language notes
- summarize friction reasons
- help structure platform and release metadata
- draft metadata for titles not yet fully modeled

### 2. Predictive Scoring

AI can:

- generate predicted affinity for unseen games
- estimate risk signals from structured metadata
- cluster titles by similarity
- draft explanation bullets for why a game may fit or fail

### 3. Ongoing Maintenance

AI can:

- update upcoming releases
- reconcile mismatched platform names
- flag missing metadata
- suggest new candidates to add to the hidden universe

## What AI Should Not Do

AI should not:

- decide whether you actually liked a game
- change statuses without a clear product rule
- invent platform availability without a source
- invent release dates
- override explicit user feedback

## What Must Remain Human

Human-owned truth:

- whether a game was enjoyed
- whether a game was abandoned
- why it failed in real life
- whether a recommendation felt right or wrong
- which tradeoffs the product makes

## Minimum Product Workflow

The clean workflow is:

1. Human provides truth
   - opinions
   - statuses
   - friction notes
   - platform access
2. AI structures and enriches that truth
3. Rule engine and scoring model generate outputs
4. Human evaluates the recommendation result
5. The system improves from that feedback loop

## Why This Matters For Portfolio

This framing makes the project more credible.

It shows that the system is not “AI because AI”.

It shows:

- where automation adds leverage
- where human data remains the source of truth
- how explainable recommendations can be built without pretending the model is infallible

## Small Product Positioning

A strong portfolio summary for this project is:

This is an AI-assisted taste engine for games. It uses structured metadata, personal play history, and explainable scoring to recommend what to continue, what to play next, and what to avoid.
